# 개인정보 보호 양식 작성 가이드 (Chrome Web Store)

개발자 대시보드의 **개인정보 보호(Privacy) 탭** 작성 시 필요한 문구입니다. 아래 양식대로 복사해서 붙여넣으세요.

---

### 1-1. 단일 목적 (단일 목적 설명)
**질문:** 이 확장 프로그램의 핵심 목적은 무엇인가요? (The single purpose of your extension)
**작성할 답변 복사:**
본 확장 프로그램(G-Cal History)의 단일 목적은 사용자가 구글 캘린더에서 새로운 일정을 생성할 때, 이전에 작성했던 유사한 과거 일정(제목, 장소, 설명)을 드롭다운으로 추천해주고, 클릭 시 자동으로 입력 칸을 채워주어 일정 등록과 캘린더 정리 시간을 획기적으로 단축시키는 캘린더 전용 생산성 도구입니다.

---

### 1-2. 권한 사용에 대한 정당성 (Permission justification)
`manifest.json`에 선언된 권한들을 왜 사용하는지 구글 심사팀에 설명해야 합니다.

**권한 1: `storage` 권한 요건 정당성**
**작성할 답변 복사:**
사용자의 캘린더 일정을 외부 서버로 전송하여 조회하지 않고, 사용자의 사용 중인 크롬 브라우저의 로컬 스토리지에 안전하게 임시 보관(데이터 캐싱)하여, 일정을 검색할 때마다 매번 구글 서버에 불필요한 API 요청을 보내는 부하를 방지하고 자동완성 팝업을 즉각적으로 띄워주기 위해 사용합니다.

**권한 2: `identity` 권한 요건 정당성 (및 Google Calendar API)**
**작성할 답변 복사:**
사용자의 구글 계정에 실제로 등록되어 있는 과거 일정 내역을 읽어와서 자동완성 추천 목록 텍스트로 보여주기 위해 오직 읽기 목적으로만 Google Calendar API(`calendar.readonly`) 인증을 요청하며, 반환된 데이터를 기반으로 신규 일정 입력 칸을 자동으로 채워주기 위한 필수 권한입니다. 데이터를 외부에 유출하거나 전송하지 않습니다.

**권한 3: 호스트 권한 사용 근거 (Host permissions justification)**
**작성할 답변 복사:**
본 확장 프로그램은 구글 캘린더 웹페이지(`https://calendar.google.com/*`) 내부에 직접 자동완성 드롭다운 UI 컴포넌트를 주입(Inject)하고, 사용자가 선택한 추천 일정 데이터를 바탕으로 웹페이지 내부의 일정 등록용 텍스트 입력 칸(Input elements)의 값을 자동으로 채워넣기 위해 해당 호스트에 대한 스크립트 실행 접근 권한이 필수적으로 요구됩니다. 또한, 과거 일정 데이터를 안전하게 불러오기 위해 Google API 서버(`https://www.googleapis.com/*`)와의 통신 호스트 권한을 사용합니다.

**질문:** 원격 코드(Remote code)를 사용하고 있습니까?
**작성할 답변:** **아니오 (사용하지 않음 / No)**

### 1-3. 데이터 사용 및 수집 공개 (Data usage and collection disclosure)
**(하단 체크박스 질문들이 나올 경우, 아래와 같이 선택하세요)**

*   **제3자에게 데이터 양도/판매 안 함:** 동의/체크(데이터를 팔지 않습니다).
*   **확장 프로그램의 핵심 목적 외의 용도로 사용자 데이터 사용 금지:** 동의/체크(핵심 기능인 일정 자동완성에만 사용합니다).
*   **신용 조회를 하거나 대출 대상 지정에 사용자 데이터 사용 금지:** 동의/체크(관련 없습니다).
