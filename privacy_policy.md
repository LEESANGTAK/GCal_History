# G-Cal History 개인정보처리방침 (Privacy Policy)

**시행일:** 2026년 2월 22일

본 개인정보처리방침은 Chrome 확장 프로그램 "G-Cal History"(이하 "본 앱")가 사용자의 개인정보를 수집, 사용, 공유 및 보호하는 방법에 대해 설명합니다. 본 앱은 개인정보 보호를 최우선으로 하며, 핵심 권한 승인을 포함하여 모든 처리 기준을 준수합니다.

---

## 1. 수집하는 정보
본 앱은 다음과 같은 정보를 명시적 목적에 한하여 제한적으로 접근 및 사용합니다.
- **Google Calendar API 데이터 (`https://www.googleapis.com/auth/calendar.readonly`)**: 자동완성 추천 목적을 위해 사용자의 캘린더에 연동된 과거 일정의 제목(Title), 장소(Location), 설명(Description), 캘린더 색상이나 분류(Calendar Summary/ID) 및 시작/종료 시간 데이터 등.
- **기본 식별 정보**: Google OAuth 2.0 시스템을 이용하기 위한 권한 부여 토큰.

## 2. 정보 사용 및 목적
본 앱은 사용자의 편의(구글 일정 팝업창에서 과거 일정 제목 자동 완성, 캘린더 자동 선택)를 제공하기 위해서만 제한적으로 데이터를 사용합니다.
1. 데이터를 브라우저 로컬 스토리지(`chrome.storage.local`)에 임시로 "캐싱"하여 반복적인 외부 API 호출을 줄이고 빠른 응답을 제공합니다.
2. 입력 창(Input Fields)과 드롭다운(Dropdown)의 UI 추천에만 그 목적을 둡니다.

## 3. 정보 공유 및 공개
본 앱은 외부 서버로 통신하는 백엔드를 가지고 있지 않습니다. 따라서 귀하의 **개인정보나 일정 데이터를 어떠한 제3자에게도 절대로 전송, 판매, 대여 또는 공유하지 않습니다.**

모든 일정 데이터 통신 과정은 당신의 브라우저(Chrome)와 Google의 공식 서버 사이에서만 직접적으로 암호화 통신(WSS/HTTPS) 형태로 발생합니다.

## 4. 정보 보호
본 앱은 데이터를 사용자의 개인 기기에 내장된 브라우저 저장소(Chrome Local Storage)에 저장합니다. 브라우저 삭제 시 본 데이터 역시 영구 삭제됩니다.

## 5. 데이터 삭제 정책
본 앱 확장을 **삭제(사용 해제)** 하는 즉시, 캐시에 남아있던 모든 구글 캘린더 일정 내용과 OAuth 인증 권한 데이터(`chrome.identity`)는 디바이스에서 완전 삭제되며 복구되지 않습니다.

## 6. 사용자의 동의
본 앱을 설치하고 OAuth 로그인 시 권한을 승인함으로써 귀하는 이 개인정보처리방침을 읽고 이해하였으며, 이 약관에 동의한 것으로 간주합니다.
